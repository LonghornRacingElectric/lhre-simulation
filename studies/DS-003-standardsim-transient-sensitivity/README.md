# DS-003 StandardSim TransientEval Sensitivity

This study is the TransientEval companion to DS-002. It reuses the compiled
Modelica variants from the DS-002 StandardSim population, so the expensive
compile-per-configuration step is not repeated.

## Question

Given the current vehicle architecture and the DS-002 parameter sweep, which
parameters most strongly move transient handling responses?

## Engine

All response results are generated by BobSim StandardSim:

`BobSim/_3_StandardSim/TransientEval/transient_eval_sim.py`

Each case points TransientEval at the existing DS-002
`build/SteadyStateEval/BobLib.Standards.VehicleSim` executable for that
variant. The directory name is historical; the executable is the same
`VehicleSim` model used by StandardSim.

## Sensitivity Definition

Each local sensitivity is one transient response variable with respect to one
varied StandardSim parameter.

- Independent variable: DS-002 parameter registry.
- Response variable: TransientEval metric listed in `outputs/metric_catalog.csv`.
- Local cases: one low-parameter run, one baseline run, one high-parameter run.
- Raw response span: `response_high - response_low`.
- Slope per unit: `(response_high - response_low) / (parameter_high - parameter_low)`.
- Normalized span: `100 * (response_high - response_low) / abs(response_baseline)`.

The active first-pass transient metrics focus on step response timing/gain,
roll/yaw response, and low-frequency FRF lag/bandwidth. Diagnostic counts,
quality metrics, and velocity-slope metrics are retained but ignored in plots.

## Run

