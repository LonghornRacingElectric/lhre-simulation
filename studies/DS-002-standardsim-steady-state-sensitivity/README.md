# DS-002 StandardSim SteadyStateEval Sensitivity

This study is the StandardSim companion to DS-001. It keeps the same
response-sensitivity framing, but every response metric comes from BobSim
StandardSim `SteadyStateEval`.

## Question

Given the current vehicle architecture in `vehicles/current`, which
StandardSim-level vehicle parameters most strongly move each steady-state
handling response variable?

## Engine

All response results are generated by BobSim StandardSim:

`BobSim/_3_StandardSim/SteadyStateEval/steady_state_eval_sim.py`

Each local parameter case writes a Modelica `variant.mo`, compiles
`BobLib.Standards.VehicleSim` with OpenModelica, runs `SteadyStateEval`, and
extracts the report metrics CSV.

Spring-rate cases also run a baseline `BobLib.Standards.FourPostSim` /
`FourPostEval` characterization first. The extracted front/rear motion ratios
are used to update spring free length for each low/high spring-rate variant, so
the spring-rate sensitivity does not accidentally include a preload/static-pose
change.

## Sensitivity Definition

Each local sensitivity is one response variable with respect to one varied
StandardSim parameter.

- Independent variable: the parameter listed in `parameter_registry.csv`.
- Response variable: the metric listed in `metric_catalog.csv`.
- Local cases: one low-parameter StandardSim build/run, one baseline build/run,
  and one high-parameter StandardSim build/run.
- Raw response span: `response_high - response_low`.
- Slope per unit: `(response_high - response_low) / (parameter_high - parameter_low)`.
- Normalized span: `100 * (response_high - response_low) / abs(response_baseline)`.

Because every local case is compiled, global sampling is optional and disabled
by default. Use `--global-samples N` when we intentionally want to spend the
extra compile budget.

Velocity-slope metrics ending in `_per_mps` are computed and retained in the raw
CSV outputs, but they are excluded from the current findings and plots. Their
baselines can be near zero, which makes percent-normalized sensitivity look
dramatic in a way that is not useful for the first design read.

`ay_min` is also retained in raw outputs but excluded from findings and plots.
It is a low-end sweep coverage metric, while `ay_max` is the lateral capability
response we care about for design.

Raw handwheel torque extrema are retained in raw outputs, but plots use
`handwheel_torque_peak_abs`, a derived response from the StandardSim torque
extrema. That keeps steering effort visible without letting near-zero
sign-crossing torque values dominate the percent-sensitivity plots.

In addition to the combined heatmap and top-sensitivity summary, the study
generates one plot per active response variable under `plots/responses/`. Those
plots are the clearest view for asking which vehicle parameters move a specific
response.

## Parameters

The first StandardSim pass varies parameters that can be coherently changed in
the Modelica vehicle record without inventing new geometry:

- front/rear spring rates
- front/rear anti-roll bar rates
- front/rear static toe
- front/rear static camber
- sprung mass, longitudinal CG, vertical CG
- body torsional stiffness
- aero downforce scale
- aero drag scale
- lateral tire friction scale
- steering rack travel per revolution


```bash
/tmp/lhre-sim-venv/bin/python studies/DS-002-standardsim-steady-state-sensitivity/run.py --limit-parameters 2
```
