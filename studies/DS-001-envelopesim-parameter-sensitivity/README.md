# DS-001 EnvelopeSim Parameter Sensitivity

This is the first fresh design study in the vehicle dynamics flow.

The goal is to locate the current vehicle architecture in the EnvelopeSim design
space before making design decisions. The study intentionally keeps response
metrics separate instead of collapsing the car into one score.

## Question

Given the current vehicle definition in `vehicles/current`, which EnvelopeSim
parameters most strongly move each response variable?

## Engine

All response results are generated by BobSim EnvelopeSim:

`BobSim/_2_EnvelopeSim/GGV/ggv_generation.py`

## Flow

1. Build a baseline `VehicleParams` object from `vehicles/current/vehicle.yml`
   and the referenced tire file.
2. Run a baseline GGV envelope.
3. Sweep every selected meaningful EnvelopeSim parameter low/base/high.
4. Compute response sensitivity for every metric independently.
5. Run a deterministic global Latin-hypercube sample to identify broader
   parameter-response correlations.
6. Generate CSV outputs, plots, and a report.

## Response Variables

For each simulated speed, the study extracts:

- `max_lateral_g`
- `max_accel_g`
- `max_brake_g`
- `ggv_area_g2`

It also computes speed-mean versions of those response variables.

## Sensitivity Definition

Each local sensitivity row is one response variable with respect to one varied
EnvelopeSim parameter.

- Independent variable: the parameter listed in `parameter_registry.csv`.
- Response variable: the metric listed in `metric_catalog.csv`.
- Local cases: one low-parameter EnvelopeSim run, one baseline run, and one
  high-parameter EnvelopeSim run.
- Raw response span: `response_high - response_low`.
- Slope per unit: `(response_high - response_low) / (parameter_high - parameter_low)`.
- Normalized span: `100 * (response_high - response_low) / abs(response_baseline)`.

The global sample is different: it reports the Pearson correlation between each
parameter and each response variable across the Latin-hypercube EnvelopeSim
sample. That correlation is a ranking tool, not a local derivative.

Because the local normalized span is computed over the selected low/high
parameter envelope, importance rankings are sensitive to the chosen parameter
bounds. Use `slope_per_unit` when the derivative-style question matters, and use
`signed_effect_pct_span` when the design-space-impact question matters.
